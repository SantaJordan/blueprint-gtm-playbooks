# Blueprint Turbo - Documentation

## Overview

Blueprint Turbo is an optimized implementation of the Blueprint GTM Intelligence System designed for **ultra-fast execution (12-15 minutes)** during live sales calls. It maintains the core quality standards of the original methodology while achieving **60-70% speed improvement** through massive parallelization and intelligent orchestration.

### Key Innovations

1. **4-Wave Parallel Execution**
   - Wave 1: 15-20 parallel intelligence gathering calls
   - Wave 2: 5-8 parallel database validation calls
   - Sequential Thinking MCP for quality synthesis
   - Browser MCP for 3x-5x faster portal scraping

2. **Live Database Discovery**
   - NO pre-built catalogs (works for ANY vertical)
   - Dynamic industry-based database search
   - Always current field names (verified from live portals)
   - Adapts automatically to healthcare, manufacturing, logistics, fintech, etc.

3. **Sales Call Optimized**
   - Real-time progress hooks (Wave 1/4, 2/4, 3/4, 4/4)
   - No approval prompts during execution
   - Predictable 12-15 minute completion
   - Immediate HTML deliverable

4. **Quality-Optimized Shortcuts**
   - 2-3 segments (vs 3-5 original) - still representative
   - 2 PQS + 2 PVP messages (vs 3+3) - still valuable
   - 1 revision cycle (vs 2) - extended thinking compensates
   - Same core standards: 8.0+/10, HIGH feasibility, actual field names

## Troubleshooting

### Issue: "Browser MCP tools not found"

**Symptom:** Command fails with "mcp__browser-mcp__chrome_navigate not available"

**Solution:**
1. Verify Browser MCP is installed: `/mcp`
2. Check Chrome/browser is running
3. Restart Claude Code if needed
4. Fallback: System will use WebFetch instead (slower but functional)

---

### Issue: "Zero HIGH feasibility segments found"

**Symptom:** Wave 2 completes but all segments rated MEDIUM/LOW/UNDETECTABLE

**Cause:** Industry databases don't support the generated segments

**Solutions:**
1. **Automatic:** System suggests 2 alternative segments, validates (+3 min)
2. **Manual:** Use top MEDIUM segment with disclaimer in HTML
3. **Expand:** Widen search to adjacent industries for more databases

---

### Issue: "All messages score <8.0 after revision"

**Symptom:** Wave 3 cannot find 4 messages that meet quality bar

**Cause:** Segments don't have strong enough insights or data is too generic

**Solutions:**
1. **Automatic:** Generate 2 additional variants, re-critique (+2 min)
2. **Quality gate:** Lower threshold to 7.5 with disclaimer in HTML
3. **Restart:** Return to synthesis, generate different segments

---

### Issue: "Execution takes >15 minutes"

**Symptom:** Command doesn't complete in target 12-15 min window

**Causes & Solutions:**
- **Slow network:** Parallel calls waiting on responses ‚Üí Use faster connection
- **Complex industry:** Many databases to research ‚Üí Acceptable 15-18 min
- **Browser MCP slow:** Chrome not responding ‚Üí Restart browser
- **Fallbacks triggered:** System degraded to WebFetch ‚Üí Accept slower time or troubleshoot MCP

---

### Issue: "Website requires JavaScript, WebFetch fails"

**Symptom:** Company website is SPA, Wave 1 WebFetch returns empty content

**Solution:** System automatically falls back to Browser MCP chrome_navigate + chrome_get_web_content (+30 sec)

## Best Practices

- ‚ùå **Deep persona research** (forums, job boards) - targeted search only

- ‚è∏ **Strategic planning sessions** - use original skills for depth

- ‚è∏ **Content marketing** - exhaustive research preferred
